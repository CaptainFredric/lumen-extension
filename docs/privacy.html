Privacy policy

Local-first capture, clear limits.

Lumen captures webpage evidence for review work. This policy describes what the current prototype handles, where it is stored, and what is not sent off-device by default.

Effective date: May 12, 2026

Default behavior

The current extension stores capture settings, redaction boxes, cutaway regions, extracted page signals, and capture history in the browser. Screenshot files and manifest files are saved through Chrome Downloads.

No advertising use

Lumen does not sell capture data, does not use capture data for personalized advertising, and does not transfer capture data for advertising.

Future sync is separate

Cloud sync, agent handoff, watch automation, billing, and team destinations are future product layers. They should require explicit user action and updated disclosure before data leaves the browser.

Information Lumen handles

The current build can handle the following data when you use capture, review, or analysis features:

  • Screenshot images of the page you choose to capture.
  • Page URL, title, host, viewport, dimensions, capture time, and export settings.
  • Redaction metadata, including detected sensitive regions and manual redaction boxes.
  • Cutaway region metadata, including selected coordinates, dimensions, and projection status.
  • Extracted page signals such as colors, fonts, navigation labels, headline text, CTA text, and layout counts.
  • Local capture history, file names, Chrome download IDs, and bundle manifest metadata.
  • Optional capture notes that you choose to add to an export.

How Lumen uses this information

Lumen uses this information to provide its user-facing capture workflow:

  • Clean sticky headers, cookie banners, chat widgets, and other page chrome before capture.
  • Generate full-page, responsive, redacted, annotated, and cutaway capture artifacts.
  • Build a JSON manifest that keeps capture context attached to the image outputs.
  • Show recent capture history and file actions in the extension popup.
  • Help you review redactions and cutaway resolution before export.

Storage and retention

Lumen stores extension settings and local history using Chrome extension storage. Screenshot images and JSON manifests are saved through Chrome Downloads into folders named by capture date. You control those files through your browser and operating system. The current build does not provide its own remote retention service.

Chrome extension storage and downloaded files are not encrypted by Lumen. Do not capture or retain sensitive pages unless you are comfortable storing those files locally.

Sharing and transfer

The current public prototype does not send screenshot content, page text, capture history, redaction metadata, or page signals to a Lumen-owned production service by default. The repository includes a local demo backend for development testing, but it is not a production sync service.

If future versions add cloud sync, agent handoff, team destinations, or scheduled watch, those features should be opt-in, visibly controlled, and described before data is sent.

Limited use disclosure

Lumen uses information received from Chrome extension APIs only to provide or improve the browser capture workflow described in the product UI and listing. Lumen does not use or transfer that information for personalized advertising, retargeting, or interest-based advertising.

The use of information received from Google APIs will adhere to the Chrome Web Store User Data Policy, including the Limited Use requirements.

Lumen does not allow human review of user capture data unless the user explicitly sends that data for support, review, or another chosen destination.

Your controls

  • You choose when to capture a page.
  • You can disable cleanup, lazy-load forcing, redaction, notes, and manifest export in the popup settings.
  • You can delete downloaded images and manifests from your Downloads folder.
  • You can clear extension storage through Chrome extension site data controls or by removing the extension.
